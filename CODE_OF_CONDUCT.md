### The Phantom-WG Principles

This document is not a list of rules with escalating penalties â€” it is a set of principles that reflect why this project exists and what it stands for.

If you are using or studying Phantom-WG, these principles are what connect us.

### Why This Matters

This is a self-hosted, open-source privacy tool. It runs on your server, under your control. There is no central authority, no user database, no telemetry, and no kill switch. That level of freedom comes with a responsibility that no enforcement mechanism can replace: your own judgment.

As described in the [Contributing Guide](CONTRIBUTING.md), this project operates under a single-maintainer model. I write every line, review every change, and take personal accountability for the code running on your server. In return, I ask that you take personal accountability for how you use it.

### Principles

**1. Privacy Is a Right, Not a Privilege**

Phantom-WG was built on the belief that privacy is a fundamental right. This means:

- Use this tool to protect your own privacy and the privacy of those who trust you.
- Never use it to violate someone else's privacy, surveil others without consent, or facilitate harassment.

**2. Freedom Requires Responsibility**

Censorship-resistant tools exist because some people need them to speak freely, access information, or simply live their lives without surveillance. This is the use case Phantom-WG is designed for.

- Do not use this project to cause harm to others.
- Do not use it for illegal activities or to facilitate illegal activities.
- The same freedoms you seek for yourself, extend to others.

**3. Respect the Humans Behind the Screen**

Whether you are opening an issue, asking a question, or discussing the project anywhere online:

- Be direct, be honest, but be respectful. You can disagree without being disagreeable.
- Criticize code, architecture, and ideas â€” not people.

**4. Knowledge Should Be Shared**

This project is fully open-source and transparent by design. The [Contributing Guide](CONTRIBUTING.md) explains the single-maintainer model and what that means in practice. In the same spirit:

- If you find a security vulnerability, report it responsibly through the process described in [SECURITY.md](SECURITY.md). Do not weaponize it.
- If you learn something useful from this codebase, share that knowledge. Write about it, teach others, build on it.
- If you fork the project, respect the AGPL-3.0 license and keep your derivative work open.

**5. Protect the Community You Are Part Of**

Phantom-WG's users include journalists, activists, researchers, and ordinary people who simply value their privacy. By using this project, you are part of that community. Do not do anything that would endanger other users or bring unwanted attention to the project.

### No Enforcement Table â€” A Word Instead

This is a self-hosted tool. I have no control over your server, nor is it possible for any sanctions to be imposed on you because of this. What I can do is maintain the spaces I control as places where these principles are respected.
